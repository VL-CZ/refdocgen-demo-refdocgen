private void DfsResolve(TNode node) #
Resolve the documentation for the selected node.
The process consists of the following steps:
  1. Select all inheritdoc elements contained in the node documentation.
  2. Resolve them one by one.
    The resolvment is done recursively using DFS and also resolves the documentation of all visited nodes.
    The end condition of recursion are:
    • The node documentation has no inheritdoc elements (i.e. nothing to resolve)
    • The node has no parent nodes (example: the type whose documentation is being resolved implements no interfaces and its base type is outside the analyzed assembly).
    Also note that the cycles are detected - the parent nodes that have already been visited are filtered out.
  3. Replace the inheritdoc elements with the resolved documentation (if there's no documentation resolved, the inheritdoc element is simply removed).
As a result, all of the 'inheritdoc' elements in the mode documentation are replaced by the actual documentation (if possible).
